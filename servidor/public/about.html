Acerca del proyecto

Objetivo General

Controlar el acceso de personas a un inmueble través de cerraduras electrónicas.

Objetivos Específicos

  1. Crear una base de datos que almacene los datos de las cerraduras, de los usuarios y de los eventos realizados en cada cerradura.
  2. Crear un servidor Node.js para gestión de datos y nodos.
  3. Diseñar una interfaz web para administrar el sistema.
  4. Construir un prototipo de cerradura con dispositivos de entrada y salida para el usuario.
  5. Conectar el sistema a una red LAN inalámbrica.
  6. Realizar pruebas para evaluar el funcionamiento del sistema.

Definición del problema

El escaso conocimiento en el área de IoT aplicada a control de acceso de personal en la Facultad Politécnica para:

  • - Generar un registro de accesos de personas a una zona específica que sea confiable y automatizado.
  • - Desarrollar un mecanismo para habilitar el paso de personas que cuentan con un libre tránsito y restringir el paso de personas no autorizadas.
  • - Acceder a los datos del sistema de forma remota utilizando la red.

Alcance

El sistema de control de acceso desarrollado cuenta con tres partes fundamentales. La primera, la cerradura, cuenta con un dispositivo de transmisión de datos y dispositivos de entrada/ salida para el usuario. La segunda, consiste en una interfaz web que posibilita a un administrador registrar: usuarios, puertas y revisar eventos. Además, permite a cada usuario registrado abrir una o más puertas según su nivel de acceso. La tercera parte consiste en una base de datos, donde se almacenan todas las informaciones del sistema.

La cerradura está compuesta por un microcontrolador, con un módulo WiFi integrado, que se encarga de almacenar y procesar las instrucciones, además posee un dispositivo de entrada de datos, el teclado matricial de 3x4 teclas, y un dispositivo de salida, el LCD 16x2 para visualizar los datos ingresados y demás datos proveídos por el sistema.

La interfaz web posee un diseño adaptativo o responsivo, está disponible para dos niveles de acceso: Administrador, que tiene permisos de crear, editar y eliminar registros de puertas y usuarios, abrir cualquier puerta registrada en el sistema desde el navegador y también de visualizar todos los eventos ocurridos en las puertas; El Encargado, tiene permisos de visualizar los usuarios registrados en el sistema, así también las puertas y además podrá abrir cualquier puerta.

La base de datos utilizada se denomina MongoDB que es una base de datos documental, además permite escalabilidad, rendimiento y gran disponibilidad.
